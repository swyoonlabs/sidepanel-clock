Why I built a clock that lives in the browser sidebar

This is the first post in a four-part series about building Side Panel Clock, a small Chrome extension I made to keep the time — local, worldwide, and focus time — one glance away. Before any code, I want to explain the itch that started it, and why the browser's Side Panel turned out to be exactly the right home for a clock.

The small annoyance that wouldn't go away

I spend most of my working day inside the browser. And all day long I kept doing the same tiny things: glancing at the clock, working out what time it was for a teammate three time zones away, and starting a quick timer so a task wouldn't quietly eat an hour.

None of these are hard. The problem is that each one pulled me away from what I was doing. The OS clock meant looking off-screen. A world-clock site meant a new tab. A timer meant yet another tab or a phone app that I'd forget to check. Individually trivial; together, a steady drip of small interruptions.

I wanted one place, always visible, that answered three questions without a context switch: what time is it here, what time is it there, and how much time do I have left?

Why not a popup, a new tab, or an app?

A browser-action popup was the obvious first idea, but popups close the moment you click anywhere else — useless for something you want to keep watching. A new-tab page only helps when you open a new tab, which is the opposite of "always visible." A separate desktop app or phone would solve the persistence problem but reintroduce the context switch I was trying to kill.

What I actually wanted was a strip of screen that stays put next to my content. Chrome shipped exactly that: the Side Panel.

💡 The Side Panel is a persistent pane docked to the side of the browser window. Unlike a popup, it stays open while you browse and rides along across tabs — perfect for a glanceable clock.

Claiming the panel: Manifest V3

Putting an extension into the Side Panel is refreshingly little configuration. You declare the sidePanel permission and point at an HTML page: 

{
  "manifest_version": 3,
  "name": "Side Panel Clock",
  "permissions": ["sidePanel", "storage", "alarms", "notifications"],
  "background": { "service_worker": "background.js" },
  "side_panel": { "default_path": "sidepanel.html" }
}

By default the panel opens from Chrome's own puzzle-piece menu. I wanted the toolbar icon to toggle it directly, which is a one-liner in the background service worker: 

chrome.runtime.onInstalled.addListener(() => {
  chrome.sidePanel
    .setPanelBehavior({ openPanelOnActionClick: true })
    .catch((err) => console.error(err));
});

// Run again at the top level so it still works
// when the service worker restarts.
chrome.sidePanel
  .setPanelBehavior({ openPanelOnActionClick: true })
  .catch((err) => console.error(err));

That second call outside the listener matters. MV3 service workers are event-driven and can be shut down at any time; setting the behavior at the top level ensures it's re-applied whenever the worker spins back up. That "the worker is not always alive" reality shaped a lot of later decisions — especially for the timer and Pomodoro, which I'll cover in this series.

One principle: one glance, no friction

With a home chosen, I set one rule for the whole project: everything answerable at a glance, nothing that demands a context switch. Concretely, that meant:

  • A large clock pinned to the top of the panel, visible no matter which sub-view you're in.
  • A world clock where you never do time-zone math — it shows the offset for you.
  • A timer and Pomodoro that keep running even when the panel is closed, and notify you.
  • No account, no sign-in, no tracking. Settings live in local storage; nothing leaves your machine.

That last point wasn't only an ethical choice, it was a simplicity choice. With no server and no analytics, there's no login flow, no privacy console, and no backend to run. The extension is just HTML, CSS, and a few hundred lines of vanilla JavaScript — no framework, no build step. For a tool this size, that keeps it fast to load and easy to reason about.
