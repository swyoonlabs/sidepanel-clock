Drawing four analog clock faces from one Canvas function

A digital readout tells you the time; an analog face gives it a shape. I wanted Side Panel Clock to have a proper analog clock — and then, because it was fun, four different faces you can switch between: classic ticks, minimal dots, Roman numerals, and plain numbers. This post walks through drawing a clock on an HTML <canvas>, and how to get four looks out of a single reusable function.

The canvas coordinate trick

Canvas puts (0, 0) in the top-left, but a clock is naturally described from its center. So the very first thing I do is translate the origin to the middle. After that, every tick and hand is just "an angle and a length from the center," which is exactly how you think about a clock.

I also pass in a size and derive a scale from it (my geometry was designed against a 300px face). That single factor lets the same code render a crisp 188px face in the panel and scale just as cleanly to any larger size, with no separate geometry to maintain. 

function renderClockFace(ctx, size, now, opts) {
  const r = size / 2;
  const scale = size / 300; // geometry designed at 300px
  const { faceBg, border, tick, dim, accent, seconds } = opts;
  const face = opts.face;          // "classic" | "minimal" | "roman" | "arabic"
  const minimal = face === "minimal";

  ctx.save();
  ctx.translate(r, r);             // origin → center

  // the dial
  ctx.beginPath();
  ctx.arc(0, 0, r - 6 * scale, 0, Math.PI * 2);
  ctx.fillStyle = faceBg;
  ctx.fill();
  ctx.lineWidth = (minimal ? 2 : 4) * scale;
  ctx.strokeStyle = border;
  ctx.stroke();
💡 Notice the function takes colors as options rather than reading them itself. That decoupling keeps the drawing independent of the theme — the caller passes whichever palette it needs, so one function serves both light and dark without any change.

One loop, four faces

The markers around the dial are the only thing that really differ between faces. A single if/else branch handles all four: 

if (face === "classic") {
  // 60 ticks; every 5th is longer and bolder
  for (let i = 0; i < 60; i++) {
    const ang = (i * Math.PI) / 30;
    const isHour = i % 5 === 0;
    const len = (isHour ? 12 : 5) * scale;
    const outer = r - 14 * scale;
    ctx.beginPath();
    ctx.lineWidth = (isHour ? 3 : 1) * scale;
    ctx.strokeStyle = isHour ? tick : dim;
    ctx.moveTo((outer - len) * Math.sin(ang), -(outer - len) * Math.cos(ang));
    ctx.lineTo(outer * Math.sin(ang), -outer * Math.cos(ang));
    ctx.stroke();
  }
} else if (minimal) {
  // 12 dots; 12/3/6/9 emphasized
  const rad = r - 18 * scale;
  for (let i = 0; i < 12; i++) {
    const ang = (i * Math.PI) / 6;
    const big = i % 3 === 0;
    ctx.beginPath();
    ctx.arc(Math.sin(ang) * rad, -Math.cos(ang) * rad, (big ? 3.5 : 2) * scale, 0, Math.PI * 2);
    ctx.fillStyle = big ? tick : dim;
    ctx.fill();
  }
} else {
  // roman / arabic numerals
  const serif = face === "roman";
  const rad = r - (serif ? 30 : 28) * scale;
  ctx.fillStyle = tick;
  ctx.font = `600 ${Math.round(r * (serif ? 0.17 : 0.2))}px ` +
    (serif ? "Georgia, serif" : "system-ui, sans-serif");
  ctx.textAlign = "center";
  ctx.textBaseline = "middle";
  for (let i = 0; i < 12; i++) {
    const ang = (i * Math.PI) / 6;
    const label = serif ? ROMAN[i] : (i === 0 ? "12" : String(i));
    ctx.fillText(label, Math.sin(ang) * rad, -Math.cos(ang) * rad);
  }
}

The angle math is the same every time: sin for the x-component, -cos for y (negative because canvas y grows downward, but 12 o'clock is up). Position i = 0 lands at the top, which is why the Roman array starts at "XII".

Hands that sweep smoothly

For the hands, the important detail is fractional time. If the hour hand only moved on the hour it would look broken at 6:30. So each hand's angle includes the smaller unit — and the second hand even folds in milliseconds for a smooth sweep: 

const h = now.getHours() % 12, m = now.getMinutes(),
      s = now.getSeconds(),   ms = now.getMilliseconds();

const secAngle  = ((s + ms / 1000) * Math.PI) / 30;
const minAngle  = ((m + s / 60)    * Math.PI) / 30;
const hourAngle = ((h + m / 60)    * Math.PI) / 6;

const hand = (angle, length, width, color) => {
  ctx.beginPath();
  ctx.lineCap = "round";
  ctx.lineWidth = width;
  ctx.strokeStyle = color;
  // extend slightly past center for a balanced look
  ctx.moveTo(-Math.sin(angle) * (length * 0.18), Math.cos(angle) * (length * 0.18));
  ctx.lineTo(Math.sin(angle) * length, -Math.cos(angle) * length);
  ctx.stroke();
};

hand(hourAngle, r * 0.5,  (minimal ? 4 : 6)   * scale, tick);
hand(minAngle,  r * 0.72, (minimal ? 2.5 : 4) * scale, tick);
if (seconds) hand(secAngle, r * 0.78, (minimal ? 1.5 : 2) * scale, accent);

The second hand is the one place the theme's accent color shows up on the dial, so it reads as a small pop against the neutral hands.

The animation loop

The live clock just clears the canvas and re-draws inside requestAnimationFrame. Because renderClockFace doesn't clear anything itself, the caller stays in control of the background — a small separation that keeps the render function reusable at any size. 

function drawAnalog(now) {
  actx.clearRect(0, 0, analog.width, analog.width);
  renderClockFace(actx, analog.width, now, faceOpts());
}
// …called every frame:
function tick() {
  drawAnalog(new Date());
  requestAnimationFrame(tick);
}

Takeaways

  • Translate to center first; describe everything as angle + length.
  • Derive a scale factor so one function renders at any size.
  • Pass colors as options — decouple drawing from theme.
  • Use fractional time so hands move continuously.
  • Let the caller clear the canvas, so the same function works for live UI and image export.
