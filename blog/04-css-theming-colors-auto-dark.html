Eight accent colors and an auto dark mode — with plain CSS variables

Side Panel Clock ships with eight accent colors and three theme modes: light, dark, and an automatic mode that follows the time of day. There's no framework and no theming library behind it — just CSS custom properties, two data- attributes, and a careful eye on specificity. This post is about making a themeable UI that stays simple.

Tokens first

Everything visual is expressed as a CSS variable on :root. Components never hard-code a color; they reference a token. Switch the tokens and the whole UI moves together. 

:root {
  --bg: #f5f6fa;
  --surface: #ffffff;
  --text: #1e293b;
  --primary: #4f46e5;   /* buttons, active tabs, ring */
  --accent: #ef4444;    /* clock second hand */
  --hero-bg: linear-gradient(160deg, #4f46e5 0%, #2563eb 100%);
}
[data-theme="dark"] {
  --bg: #0f172a;
  --surface: #1e293b;
  --text: #e2e8f0;
  --primary: #6366f1;
  --accent: #f87171;
  --hero-bg: linear-gradient(160deg, #312e81 0%, #1e3a8a 100%);
}

Dark mode is just a second set of the same tokens, activated by a data-theme="dark" attribute on the root element. So far, standard stuff.

Accent colors: a second, orthogonal axis

The interesting problem is that theme (light/dark) and accent (indigo, teal, rose…) are independent. Any of the eight colors must work in either mode. I represent the accent with a separate data-accent attribute and define palettes for it — but the ordering interacts with dark mode in a way you have to get right. 

/* light accent: :root[data-accent] beats plain :root */
:root[data-accent="teal"]  { --primary:#0d9488; --accent:#f59e0b;
  --hero-bg: linear-gradient(160deg,#14b8a6,#0f766e); }

/* dark accent: must also beat the [data-theme=dark] block */
:root[data-theme="dark"][data-accent="teal"] { --primary:#2dd4bf; --accent:#fbbf24;
  --hero-bg: linear-gradient(160deg,#115e59,#134e4a); }
⚠️ The specificity gotcha. A selector like :root[data-accent="teal"] has higher specificity than the plain [data-theme="dark"] block. So if you only defined the light accent, picking an accent in dark mode would drag the light colors back in. The fix is to always pair each accent with a dark variant using both attributes — that two-attribute selector wins in dark mode, and the single-attribute one wins in light mode. Indigo needs no block at all; it's the default that the base tokens already describe.

Applying an accent from JavaScript is then a single attribute write:

function applyAccent() {
  document.documentElement.setAttribute("data-accent", settings.accent);
}

Because the analog clock reads --accent from the computed styles each frame (see Part 3), the second hand recolors instantly with no extra wiring.

Three theme modes, including "auto"

The theme toggle cycles through three states. "Auto" is the one with logic: it picks light or dark based on the current hour. 

const THEME_CYCLE = ["light", "dark", "auto"];

function autoTheme(now = new Date()) {
  const h = now.getHours();
  return h >= 7 && h < 19 ? "light" : "dark";  // 07:00–19:00 = light
}
function effectiveTheme() {
  return settings.theme === "auto" ? autoTheme() : settings.theme;
}
function applyTheme() {
  document.documentElement.setAttribute("data-theme", effectiveTheme());
}

The subtlety with "auto" is that it can change while the panel is open. So in the same animation loop that ticks the clock, I re-apply the theme only when the day/night boundary is actually crossed — cheap, and no flicker: 

if (settings.theme === "auto" &&
    document.documentElement.getAttribute("data-theme") !== autoTheme(now)) {
  applyTheme();
}

(If you'd rather follow the operating system instead of the clock, the same structure works — just read matchMedia("(prefers-color-scheme: dark)") in place of autoTheme.)

Why not a framework?

It's tempting to reach for a theming library, but for a UI this size CSS variables are the theming engine. The entire system is: define tokens, flip two attributes, let the cascade do the rest. No re-render, no context providers, no runtime style objects. The clock canvas participates too, simply by reading the same computed variables. It's the kind of solution that keeps working precisely because there's so little of it.

Takeaways

  • Express all colors as tokens on :root; components reference them.
  • Treat theme and accent as independent axes via two data- attributes.
  • Watch specificity: pair every accent with a two-attribute dark variant.
  • An "auto" mode is just a function choosing light/dark — by clock or by OS.
